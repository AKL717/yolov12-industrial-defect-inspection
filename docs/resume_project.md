# 简历项目写法

## 项目名称

基于 C++/OpenCV/ONNX Runtime 的工业视觉缺陷检测与定位系统

## 项目描述

面向工业质检场景，设计并实现一套缺陷检测与定位系统。系统支持图片、文件夹批量检测、视频及摄像头输入，基于 OpenCV 完成图像预处理、阈值分割、形态学滤波、轮廓检测、缺陷候选区域提取和几何特征筛选，并接入 ONNX Runtime 实现 YOLO 模型推理。系统支持 traditional、onnx、hybrid 三种检测模式，能够读取 YOLO 格式真实标注，生成预测框与真实框对比图，并输出 TP/FP/FN、Precision、Recall、F1、IoU、缺陷面积和耗时等 CSV/文本报表。

## 可写亮点

- 搭建 C++ 工业视觉检测工程框架，完成配置读取、图像输入、检测、可视化、结果保存等模块化设计。
- 使用 CLAHE、GaussianBlur、Otsu/自适应阈值、形态学开闭运算等传统图像算法增强缺陷区域。
- 基于轮廓检测提取缺陷候选区域，并通过面积、长宽比、圆度等特征进行误检过滤。
- 实现实时检测结果绘制，输出缺陷数量、外接矩形、中心点、面积、OK/NG 判定及 FPS。
- 实现批量图片质检流程，按 OK/NG 分类保存检测结果，并生成包含缺陷数量、最大面积、总面积和耗时的 CSV 报表。
- 接入 NEU-DET 钢材表面缺陷数据集，支持读取 YOLO 标签并生成预测框/真实框对比图，统计 TP、FP、FN 和平均匹配 IoU。
- 编写一键构建、验证集评估和发布打包脚本，输出包含可执行程序、OpenCV DLL、配置文件、样例数据和说明文档的发布包。
- 基于 ONNX Runtime C++ API 实现 YOLO 模型推理后端，完成 letterbox 预处理、Tensor 构建、模型推理、输出解析、NMS 和坐标还原。
- 支持 traditional、onnx、hybrid 三种检测模式，将传统视觉和深度学习检测流程统一到同一套报表与可视化框架中。
- 在 RTX 2060 上完成 YOLOv12n + NEU-DET 训练链路验证，训练 50 epoch 后导出 ONNX 模型并接入 C++ 程序；ONNX Runtime 部署模型在验证集上达到 Precision 0.846、Recall 0.859、F1 0.853、平均匹配 IoU 0.738，显著优于传统视觉 baseline。

## 面试讲解思路

1. 先说明工业视觉项目不能只依赖模型，传统预处理和稳定的工程流程很重要。
2. 介绍图像从输入到 OK/NG 判定的完整流程。
3. 说明为什么要用形态学、连通域/轮廓分析和特征筛选。
4. 展示配置文件如何快速适配不同产品和光照。
5. 最后补充后续可接入 ONNX Runtime，把 YOLO 作为复杂缺陷的检测模块。
